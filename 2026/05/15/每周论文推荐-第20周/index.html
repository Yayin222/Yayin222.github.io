本周精选论文

本周共收集 102 篇新论文，精选 15 篇重点关注方向，涵盖 Agent 安全、提示注入防御、Skill 生态安全、内存机制等主题。

1. Self-ReSET: Learning to Self-Recover from Unsafe Reasoning Trajectories

来源: arXiv 2605.08936
链接: https://arxiv.org/abs/2605.08936
关键词: reasoning, safety, self-recovery, RL, jailbreak
核心思想: 提出一种让 LLM 在推理过程中检测到不安全轨迹后，能够自主恢复到安全状态的机制，而非简单拒绝。通过强化学习训练模型学会”自我纠错”，在保持有用性的同时提升安全性。

2. AgentForesight: Online Auditing for Early Failure Prediction in Multi-Agent Systems

来源: arXiv 2605.08715
链接: https://arxiv.org/abs/2605.08715
关键词: multi-agent, safety, failure prediction, auditing
核心思想: 在多 Agent 系统中引入在线审计机制，通过实时监控 Agent 行为轨迹，在故障发生前进行预测和干预。解决了多 Agent 协作中”事后才发现问题”的痛点。

3. Five Attacks on x402 Agentic Payment Protocol

来源: arXiv 2605.11781
链接: https://arxiv.org/abs/2605.11781
关键词: web security, agent, payment, vulnerability, blockchain
核心思想: 对 x402 Agentic 支付协议进行安全分析，发现并披露了五种攻击向量。随着 AI Agent 开始自主执行金融交易，支付协议的安全性成为关键问题。本文为 Agentic 支付生态的安全设计提供了重要参考。

4. IPI-proxy: An Intercepting Proxy for Red-Teaming Web-Browsing AI Agents Against Indirect Prompt Injection

来源: arXiv 2605.11868
链接: https://arxiv.org/abs/2605.11868
关键词: prompt injection, web security, agent, red-teaming, proxy
核心思想: 构建了一个拦截代理工具，专门用于对浏览网页的 AI Agent 进行间接提示注入的红队测试。能够系统性地在网页内容中嵌入注入 payload，测试 Agent 的抗注入能力。实用工具性很强。

5. Behavioral Integrity Verification for AI Agent Skills

来源: arXiv 2605.11770
链接: https://arxiv.org/abs/2605.11770
关键词: agent, skill, integrity, verification, OpenClaw
核心思想: 提出对 AI Agent Skill 的行为完整性验证机制，确保 Skill 实际执行的行为与声明的功能一致。直接关联 OpenClaw 等 Agent 框架的 Skill 生态安全。

6. Proteus: A Self-Evolving Red Team for Agent Skill Ecosystems

来源: arXiv 2605.11891
链接: https://arxiv.org/abs/2605.11891
关键词: agent, skill, red team, security, adaptive attack
核心思想: 构建了一个能够自演化的红队系统，专门针对 Agent Skill 生态进行持续性安全测试。红队能够根据防御反馈自适应调整攻击策略，模拟真实世界中的高级威胁。

7. Comment and Control: Hijacking Agentic Workflows via Context-Grounded Evolution

来源: arXiv 2605.11229
链接: https://arxiv.org/abs/2605.11229
关键词: agent, agentic, exploit, LLM
核心思想: 揭示了一种通过代码注释等上下文信息劫持 Agentic 工作流的攻击方式。攻击者可以通过精心构造的上下文内容，让 Agent 在执行工作流时被”隐形”引导至恶意行为。对 CI/CD 中的 Agent 自动化流程构成严重威胁。

8. FlowSteer: Prompt-Only Workflow Steering Exposes Planning-Time Vulnerabilities in Multi-Agent LLM Systems

来源: arXiv 2605.11514
链接: https://arxiv.org/abs/2605.11514
关键词: planning, defense, LLM, vulnerability, multi-agent
核心思想: 发现多 Agent LLM 系统在规划阶段存在漏洞——仅通过提示注入就能操控整个工作流的执行方向。攻击发生在规划时而非执行时，使得传统的运行时防御难以检测。

9. Sleeper Channels and Provenance Gates: Persistent Prompt Injection in Always-on Autonomous AI Agents

来源: arXiv 2605.13471
链接: https://arxiv.org/abs/2605.13471
关键词: prompt injection, agent, OpenClaw, persistent, sleeper channel
核心思想: 针对”常驻型”自主 AI Agent（如 OpenClaw），提出”休眠通道”概念——注入的恶意提示可以长期潜伏，在特定条件下激活。同时提出”来源门”（Provenance Gates）作为防御机制，通过验证数据来源来阻断持久化注入。

10. No Attack Required: Semantic Fuzzing for Specification Violations in Agent Skills

来源: arXiv 2605.13044
链接: https://arxiv.org/abs/2605.13044
关键词: fuzzing, agent, specification, LLM, skill
核心思想: 不需要构造攻击，通过语义模糊测试就能发现 Agent Skill 中的规范违反问题。使用 LLM 生成多样化的输入场景，自动发现 Skill 行为不符合预期规范的情况。对 Skill 开发者是实用的质量保障工具。

11. SkillSafetyBench: Evaluating Agent Safety under Skill-Facing Attack Surfaces

来源: arXiv 2605.12015
链接: https://arxiv.org/abs/2605.12015
关键词: adversarial, safety, agent, benchmark, attack
核心思想: 提出一个专门针对 Agent Skill 攻击面的安全评估基准。系统性地定义了 Skill 层面的攻击面，并提供了标准化的评估方法，填补了 Agent Skill 安全评估的空白。

12. From Storage to Experience: A Survey on the Evolution of LLM Agent Memory Mechanisms

来源: arXiv 2605.06716 (ACL 2026)
链接: https://arxiv.org/abs/2605.06716
关键词: agent, LLM, memory, survey, ACL
核心思想: ACL 2026 收录的综述，系统梳理了 LLM Agent 记忆机制的演进——从简单的存储到经验化的记忆系统。涵盖了短期记忆、长期记忆、情景记忆等多种记忆范式，以及它们在 Agent 架构中的应用。对理解 Agent 记忆系统设计非常有参考价值。

13. The Authorization-Execution Gap Is a Major Safety and Security Problem in Open-World Agents

来源: arXiv 2605.11003
链接: https://arxiv.org/abs/2605.11003
关键词: defense, safety, agent, autonomous, multi-agent
核心思想: 指出开放世界 Agent 面临的核心安全问题——授权与执行之间的鸿沟。Agent 可能获得超出预期的权限并执行未经授权的操作，提出需要建立更精细的授权-执行映射机制。

14. ExploitGym: Can AI Agents Turn Security Vulnerabilities into Real Attacks?

来源: arXiv 2605.11086
链接: https://arxiv.org/abs/2605.11086
关键词: reasoning, defense, exploit, vulnerability, agent, benchmark
核心思想: 构建了一个让 AI Agent 尝试将安全漏洞转化为实际攻击的基准环境。测试 Agent 的漏洞利用推理能力，同时也为防御方提供了评估 Agent 安全能力的标准化框架。

15. Red-Teaming Agent Execution Contexts: Open-World Security Evaluation on OpenClaw

来源: arXiv 2605.11047
链接: https://arxiv.org/abs/2605.11047
关键词: adversarial, vulnerability, agent, benchmark, red-teaming, OpenClaw
核心思想: 在 OpenClaw 框架上进行开放世界安全评估，红队测试 Agent 的执行上下文。系统性地评估了 Agent 在真实部署环境中的安全边界，为 Agent 框架的安全加固提供了实证依据。

本周趋势观察

  1. Agent Skill 安全成为焦点: 多篇论文聚焦 Skill 层面的安全问题（SkillScope、Proteus、SkillSafetyBench、Semantic Fuzzing），表明社区正在从”Agent 整体安全”深入到”组件级安全”。

  2. 持久化注入攻击升级: Sleeper Channels、Comment and Control 等工作揭示了更隐蔽的注入方式，从单次攻击演变为长期潜伏式攻击。

  3. 红队自动化: Proteus、ExploitGym、Cochise 等工作推动了红队测试的自动化和标准化，Agent 安全测试正从手工走向系统化。

  4. 授权与治理: Authorization-Execution Gap、Digital Identity、MCP Authorization 等论文关注 Agent 的身份和权限治理，这是 Agent 走向生产部署的必经之路。
