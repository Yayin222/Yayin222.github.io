各大厂商 Agent 论文精选指南：从 Anthropic 到 Google DeepMind

发表于2026-05-14|更新于2026-05-14|AI Agent
|浏览量:

2025-2026 年是 AI Agent 爆发的元年。各大厂商纷纷发布重磅论文和技术文章，从架构设计到工程实践，从单 Agent 到多 Agent 协作，Agent 领域正在快速演进。本文精选了最值得学习的论文和技术报告，按厂商分类整理。

🔵 Anthropic (Claude)

Anthropic 在 Agent 方向的输出非常系统化，从设计理念到工程实践都有覆盖，是学习 Agent 工程化的最佳起点。

1. Building Effective Agents ⭐ 必读

链接: anthropic.com/research/building-effective-agents

核心思想: Anthropic 的 Agent 架构设计哲学。提出了从简单到复杂的渐进式设计模式：

  • Augmented LLM → Workflow → Agent 三层递进
  • 工具优化比 prompt 优化更重要——在 SWE-bench 项目中，他们花在优化工具上的时间比优化 prompt 更多
  • 不要过度设计，能用简单 workflow 解决的问题就不要上 Agent

适合人群: 想理解 Agent 工程化设计思路的开发者

2. Effective Context Engineering for AI Agents

链接: anthropic.com/engineering/effective-context-engineering-for-ai-agents

核心思想: Agent 上下文管理的核心策略。随着 Agent 运行时间变长，context window 管理成为关键瓶颈：

  • Compaction（压缩）: 将长对话历史压缩为摘要
  • Tool Result Clearing: 清理已处理的工具返回结果
  • Memory Tool: 基于文件系统的外部记忆，让 Agent 持久化存储信息

适合人群: 做长时间运行 Agent 的开发者

3. How We Built Our Multi-Agent Research System

链接: anthropic.com/engineering/multi-agent-research-system

核心思想: Claude Research 的多 Agent 架构实战。采用 LeadResearcher + 并行 Subagent 模式：

  • LeadResearcher 负责规划研究流程
  • 多个 Subagent 并行搜索信息，独立评估结果
  • 使用 Interleaved Thinking 让 Agent 在工具调用间隙进行推理

适合人群: 对多 Agent 协作架构感兴趣的人

4. Writing Effective Tools for AI Agents

链接: anthropic.com/engineering/writing-tools-for-agents

核心思想: 为 Agent 设计工具和为人类设计 API 是完全不同的思路：

  • 工具描述要面向 Agent 而非人类开发者
  • 可以用 Agent 自己来优化工具——让 Claude Code 协助改进 MCP Server
  • 好的工具设计 = 好的 Agent 性能

🟢 OpenAI

OpenAI 的 Agent 路线更偏向产品化落地，从 Deep Research 到 Codex，展示了 Agent 在不同场景的工程实践。

5. Harness Engineering: Leveraging Codex in an Agent-First World ⭐ 必读

链接: openai.com/index/harness-engineering

核心思想: OpenAI 内部用 Codex 以 0 手写代码 构建产品的经验总结。核心洞察：

  • 人类角色从「写代码」转向「设计环境、描述意图、构建反馈循环」
  • 效率提升约 10 倍
  • 新的开发范式：Harness Engineering——设计让 Agent 高效工作的环境

关键概念:

  • 环境设计 > 代码编写
  • 反馈循环 > 一次性 prompt
  • 可观测性 > 事后调试

适合人群: 想理解 Agent-first 开发模式的工程师

6. OpenAI Agents SDK

链接: developers.openai.com/api/docs/guides/agents

核心思想: OpenAI 的 Agent 开发框架，包含几个关键设计：

  • Guardrails: 输入输出安全护栏
  • Handoffs: Agent 间的任务交接机制
  • Tracing: 全链路可观测性
  • Compaction: 上下文压缩策略

7. Deep Research & Operator

Deep Research: openai.com/index/introducing-deep-research
Operator: openai.com/index/introducing-operator

核心思想: 两种不同的 Agent 范式：

特性 Deep Research Operator
交互模式 深度研究，自主搜索+分析+生成报告 浏览器操控，GUI Agent
核心能力 推理 + 信息综合 视觉理解 + 操作
适用场景 研究报告、数据分析 网页操作、表单填写
底层模型 o3 优化版 GPT-4o + 视觉

🟡 Google DeepMind

Google 在 Agent 方向更偏学术和系统性研究，论文质量很高。

8. Towards a Science of Scaling Agent Systems ⭐ 必读

链接: research.google/blog/towards-a-science-of-scaling-agent-systems

核心思想: 通过 180 种 Agent 配置 的系统实验，得出多 Agent 系统的量化扩展原则：

  • 可并行任务: 多 Agent 效果显著提升
  • 顺序任务: 多 Agent 反而退化
  • 引入预测模型，为 87% 的未见任务选择最优架构

关键发现:

  • 不是所有任务都适合多 Agent
  • 架构选择可以被量化预测
  • Agent 数量存在最优解

适合人群: 想从科学角度理解 Agent 系统扩展性的研究者

9. ReasoningBank: Enabling Agents to Learn from Experience

链接: research.google/blog/reasoningbank-enabling-agents-to-learn-from-experience

核心思想: 让 Agent 从经验中学习推理策略，实现持续进化而非每次从零开始。

10. Gemini 2.0 & Agent 生态

链接: blog.google/gemini-2.0

核心思想: Google 的完整 Agent 生态布局：

  • Project Mariner: 浏览器 Agent
  • A2A Protocol: Agent 间通信标准（对标 Anthropic 的 MCP）
  • ADK (Agent Development Kit): Agent 开发工具包
  • Gemini Robotics: 具身智能 Agent

📚 综合 Survey & Benchmark

11. The 2025 AI Agent Index (MIT)

链接: aiagentindex.mit.edu

系统性梳理了所有主流 Agent 产品、安全措施、能力评估。适合全面了解 Agent 产业现状。

12. Evaluation and Benchmarking of LLM Agents: A Survey (KDD’25)

Agent 评估基准的综合综述，涵盖通用 Agent、多 Agent、安全评估等多个维度。

13. TheAgentCompany (NeurIPS 2025)

链接: openreview.net/forum?id=LZnKNApvhG

模拟真实工作环境的 Agent 基准测试，评估 Agent 处理邮件、写代码、与同事沟通等能力。结论：简单任务可自主完成，复杂长程任务仍是挑战。

💡 学习路径建议

如果你时间有限，推荐按以下顺序阅读：

入门级（3 篇）

  1. Anthropic “Building Effective Agents” — 理解 Agent 设计哲学
  2. OpenAI “Harness Engineering” — 理解 Agent-first 开发实践
  3. Google “Towards a Science of Scaling Agent Systems” — 理解 Agent 系统的科学原理

进阶级（+3 篇）

  1. Anthropic “Effective Context Engineering” — 上下文管理
  2. Anthropic “Multi-Agent Research System” — 多 Agent 架构
  3. Google “ReasoningBank” — Agent 经验学习

深入级（+2 篇）

  1. OpenAI Agents SDK 文档 — 工程实现
  2. TheAgentCompany Benchmark — 评估方法论

📊 各厂商 Agent 策略对比

维度 Anthropic OpenAI Google
核心理念 渐进式设计，工具优先 Agent-first，产品驱动 学术驱动，系统性研究
Agent 范式 Workflow → Agent Deep Research / Codex / Operator A2A 协议 + ADK
多 Agent LeadResearcher + Subagent Codex 团队协作 量化扩展原则
工具标准 MCP Agents SDK A2A Protocol
开源程度 高（Cookbook + MCP） 中（SDK + 文档） 中（ADK + 论文）

写在最后: Agent 领域正在从「能不能用」走向「怎么用好」。各大厂商的思路虽然不同，但核心共识是：好的 Agent = 好的工具设计 + 好的上下文管理 + 好的反馈循环。这比单纯追求更大的模型更有意义。
