本周精选论文

本周共收集 21 篇 论文，涵盖漏洞挖掘、Agent 系统、LLM 安全与推理等方向。

🔒 安全方向（13 篇）

1. The Misattribution Gap: When Memory Poisoning Looks Like Model Failure in Agentic AI Systems

来源: arXiv (2605.22842)
链接: https://arxiv.org/abs/2605.22842
核心思想: 提出了 Semantic Norm Drift 攻击——攻击者通过污染共享向量存储中的文档，使 AI Agent 产生与模型故障无法区分的错误行为。与传统数据投毒关注模型训练不同，该攻击瞄准 Agent 运行时的检索增强生成（RAG）流程。论文提出了 Counterfactual Composition Testing 和 Memory-Persistent Information-Flow Control 两种防御机制。

标签: Agent安全 RAG投毒 向量存储

2. PoisonForge: Task-Level Targeted Poisoning Benchmark for Instruction-Tuned LLMs

来源: arXiv (2605.23168)
链接: https://arxiv.org/abs/2605.23168
核心思想: 构建了参数化的 LLM 数据投毒基准测试，沿 4 个维度评估 12 个模型。惊人的发现是：仅需 1% 的投毒预算（10 个投毒样本），12 个模型中有 11 个的攻击成功率超过 70%。研究还建立了预测模型，揭示投毒设计选择对攻击成功率的关键影响。

标签: 数据投毒 LLM安全 基准测试

3. Prompt Overflow: What the Guardrail Inspects Is Not What the Model Infers

来源: arXiv (2605.23196)
链接: https://arxiv.org/abs/2605.23196
核心思想: 发现了一种新颖的攻击方式——利用护栏模型有限的检查窗口与 LLM 更大上下文窗口之间的不匹配。攻击者将恶意指令碎片化并穿插在良性填充文本中，成功绕过了 Prompt Guard、Granite Guardian、DeBERTa 等最先进的护栏模型。揭示了当前安全对齐机制的根本性缺陷。

标签: Prompt注入 护栏绕过 对齐

4. Are Frontier LLMs Ready for Cybersecurity? Evidence for Vertical Foundation Models

来源: arXiv (2605.23243)
链接: https://arxiv.org/abs/2605.23243
核心思想: 在双模式漏洞基准测试中评估了 6 个前沿 LLM（GPT-5.4、Codex 5.3、Claude、Gemini）。白盒漏洞检测表现尚可，但在黑盒 Web 安全测试中，前沿模型仅能覆盖 4-8% 的真实漏洞。论文认为通用 LLM 在网络安全领域存在根本性局限，需要专门构建垂类基础模型。

标签: 网络安全 LLM评估 垂类模型

5. CachePrune: Privacy-Aware and Fine-Grained KV Cache Sharing for Efficient LLM Inference

来源: arXiv (2605.23640)
链接: https://arxiv.org/abs/2605.23640
核心思想: 提出了 Token 级别的隐私感知 KV Cache 共享机制，消除了缓存重用中的侧信道泄露。在保证隐私的前提下，TTFT（首 Token 延迟）降低了 4.5 倍，缓存命中率相比 SOTA 提升了 44%。解决了 LLM 推理效率与隐私保护之间的核心矛盾。

标签: LLM推理 隐私保护 KV缓存

6. Security, Privacy, and Ethical Risks in OpenClaw

来源: Journal of Information and Intelligence (JII) (2605.23330)
链接: https://arxiv.org/abs/2605.23330
核心思想: 系统性地调查了本地可执行 AI Agent 系统中的安全、隐私和伦理风险。涵盖了持久化本地存储、工具调用、跨上下文数据聚合、多用户交互和插件集成等维度的风险分析。为 Agent 系统的安全设计提供了全面的威胁模型。

标签: Agent安全 隐私 伦理

7. What Does the Server See? Understanding Privacy Leakage from LLMs in Split Inference

来源: ACM CCS’26 (2605.23158)
链接: https://arxiv.org/abs/2605.23158
核心思想: 提出了 ActInv 攻击，可从 LLM 分割推理的中间层表示中重建客户端输入。引入了 PAF 指标量化不同层的隐私泄露脆弱性，并提出了 PriPert 防御方案。该工作已被 CCS’26 接收，对端侧-云端协同推理的隐私保护有重要启示。

标签: 分割推理 隐私泄露 CCS'26

8. FuzzingBrain V2: A Multi-Agent LLM System for Automated Vulnerability Discovery and Reproduction

来源: arXiv (2605.21779)
链接: https://arxiv.org/abs/2605.21779
核心思想: 多 Agent LLM 系统用于自动化漏洞发现。使用 Suspicious Point 抽象和双层模糊测试策略。在 AIxCC 2025 上达到 90% 的检出率，在 12 个真实项目中发现了 29 个零日漏洞，其中 2 个获得 CVE 编号。展示了多 Agent 协作在漏洞挖掘中的巨大潜力。

标签: Fuzzing 多Agent 漏洞挖掘

9. Quality-Assured Fuzz Harness Generation via the Four Principles Framework

来源: arXiv (2605.21824)
链接: https://arxiv.org/abs/2605.21824
核心思想: 提出了 QuartetFuzz 系统，基于四大原则（逻辑正确性、API 协议、安全边界、入口点）生成高质量的 Fuzz Harness。在 23 个项目中发现了 29 个 Bug 和 3 个 CVE，并发现 70 个项目的现有 Harness 中存在 53 处违规。为自动化 Fuzzing 的质量保证提供了系统化框架。

标签: Fuzzing Harness生成 质量保证

10. Latent-space Attacks for Refusal Evasion in Language Models

来源: arXiv (2605.21706)
链接: https://arxiv.org/abs/2605.21706
核心思想: 将 LLM 的拒绝抑制重新定义为针对线性探针的潜在空间逃逸攻击。提出了 Controlled Latent-space Evasion 攻击，通过将表征投影到决策边界之外来绕过安全拒绝机制。在 15 个模型上达到了 SOTA 攻击成功率，揭示了当前安全对齐在表征层面的脆弱性。

标签: 拒绝绕过 潜在空间 对齐安全

11. Building an Open Source OT Pentesting Platform: Lessons from LINICS

来源: IEEE Security & Privacy (2605.22590)
链接: https://arxiv.org/abs/2605.22590
核心思想: 分享了构建和发布 LINICS 的经验——首个面向 OT/ICS（工控系统）渗透测试的开源平台。论文发表在 IEEE Security & Privacy，详细讨论了工控安全工具化的架构设计、工程挑战和开源社区建设经验。

标签: OT安全 工控 渗透测试

12. Measuring Security Without Fooling Ourselves: Why Benchmarking Agents Is Hard

来源: arXiv (2605.22568)
链接: https://arxiv.org/abs/2605.22568
核心思想: 揭示了 AI Agent 安全评估面临的三大核心挑战：基准数据集的脆弱性、时效性退化、和运行时不确定性。论文提出了构建鲁棒评估框架的实用方向，指出当前的 Agent 安全基准可能在”自己骗自己”。

标签: 安全评估 基准测试 Agent安全

13. ASSEMBLAGE-DEEPHISTORY: A Cross-Build Binary Dataset with Temporal Coverage

来源: arXiv (2605.21615)
链接: https://arxiv.org/abs/2605.21615
核心思想: 发布了包含 73,610 个二进制文件、覆盖 248 个项目多年演化历史的数据集。这是首个结合跨构建多样性、版本历史和 CVE 标签的数据集。配套提供了面向二进制漏洞推理的 LLM 基准测试，为二进制安全研究提供了宝贵的资源。

标签: 二进制安全 数据集 漏洞推理

🤖 Agent 与推理方向（8 篇）

14. When Do LLMs Reason? A Dynamical Systems View via Entropy Phase Transitions

来源: arXiv (2605.22873)
链接: https://arxiv.org/abs/2605.22873
核心思想: 从动力系统视角揭示了 LLM 推理是一个可通过早期熵动态检测的解码状态。提出了 EDRM 框架，通过自适应选择 CoT 推理策略，在 15 个基准测试上实现了 41-55% 的 Token 减少，同时提升了准确率。为理解 LLM 何时”真正在推理”提供了理论洞察。

标签: 推理机制 熵动态 CoT优化

15. Latent Cache Flow: Model-to-Model Communication Without Text

来源: arXiv (2605.22863)
链接: https://arxiv.org/abs/2605.22863
核心思想: 提出了一种突破性的方法：LLM Agent 之间通过压缩的 KV Cache 交换而不是文本来通信。相比基于文本的通信，准确率提升了 23%，速度提升 8.5 倍。仅需 13MB 适配器就达到了优于 956MB 先前工作的性能。专为异构 Agent 上下文设计，可能重新定义多 Agent 系统的通信范式。

标签: Agent通信 KV缓存 多Agent

16. MARICL: From Residuals to Reasons - LLM-Guided Mechanism Extraction from Multi-Agent Simulators

来源: arXiv (2605.22897)
链接: https://arxiv.org/abs/2605.22897
核心思想: 提出了 Multi-Agent Residual In-Context Learning 框架，LLM Agent 通过分析模型残差并通过文本梯度优化生成显式修正项。在生化实验批次之间展示了机制泛化能力，为科学领域的多 Agent 建模提供了新思路。

标签: 多Agent 残差学习 科学计算

17. Planning in the LLM Era: Building for Reliability and Efficiency

来源: ICAPS 2026 (2605.21902)
链接: https://arxiv.org/abs/2605.21902
核心思想: 发表在 ICAPS 2026，主张从单次 LLM 规划向 LLM 生成符号规划器转变。涵盖了三种规划器生成方法，论证了混合方法（LLM 负责生成、符号系统负责执行）在可靠性和效率上显著优于纯 LLM 规划。

标签: 规划 符号推理 ICAPS'26

18. MindLoom: Composing Thought Modes for Frontier-Level Reasoning Data Synthesis

来源: arXiv (2605.21630)
链接: https://arxiv.org/abs/2605.21630
核心思想: 通过组合式思维模式工程合成前沿级推理数据的框架。将解决方案分解为思维模式链，训练检索模型进行引导式合成。在 5 个 STEM 学科的 9 个基准测试上进行了评估，为高质量的推理训练数据生成开辟了新路径。

标签: 推理合成 思维模式 训练数据

19. Trace2Skill: Verifier-Guided Skill Evolution for Long-Context EDA Agents

来源: arXiv (2605.21810)
链接: https://arxiv.org/abs/2605.21810
核心思想: 提出了测试时扩展框架，将 Agent 技能视为可进化的策略。从 rollout 轨迹中挖掘成功和失败模式，实现了技能的自适应进化。在 CVDP 硬件设计任务上取得了突破性进展，解决了此前无法完成的任务——而且无需模型微调。

标签: EDA 技能进化 测试时扩展

20. Benchmarking and Improving Monitors for OOD Alignment Failure in LLMs

来源: arXiv (2605.21602)
链接: https://arxiv.org/abs/2605.21602
核心思想: 引入 MOOD 基准测试，用于检测 LLM 的分布外（OOD）对齐失败。发现护栏模型无法泛化到 OOD 场景。提出了将护栏模型与 OOD 检测器结合的方法，将召回率从 39% 提升到 45%，但仍远未解决 OOD 场景下的安全对齐问题。

标签: OOD检测 对齐 安全监控

21. TO-Agents: A Multi-Agent AI Pipeline for Preference-Guided Topology Optimization

来源: ASME IDETC 2026 (2605.21622)
链接: https://arxiv.org/abs/2605.21622
核心思想: 多 Agent 框架连接自然语言设计意图与迭代拓扑优化。使用多视图 VLM 推理配合独立的评判 Agent。已被 ASME IDETC 2026 接收，展示了多 Agent 系统在工程设计优化中的应用潜力。

标签: 多Agent 拓扑优化 工程设计
